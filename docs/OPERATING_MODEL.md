# Operating Model

## Intent

This repository is meant to make a communications function legible, repeatable, and compounding. It should help an operator move faster without becoming more generic.

## Two-speed system

### Daily heartbeat
Use the daily heartbeat to reduce surprise and improve interaction quality.

Core loop:
1. Read the morning picture with `/landscape`.
2. Prepare for interactions with `/brief`.
3. Update relationship memory with `/rolodex`.

### Campaign surges
Use campaign surges for launches, data franchises, narrative pushes, founder stories, and amplification.

Canonical loop:
1. Define the thesis with `/narrative`.
2. Convert it into hooks with `/angles`.
3. Design execution with `/campaign`.
4. Draft support materials with `/draft`.
5. Prepare the humans with `/brief`.
6. Amplify or package with `/linkedin`, `/founder-voice`, and `/data` as needed.
7. Measure outcomes with `/monitor`.
8. Learn with `/retro`.

### Emergency mode
Use emergency mode when uncertainty, speed, and stakeholder complexity collide.

Canonical loop:
1. Detect with `/landscape`.
2. Open the incident with `/warroom`.
3. Package stakeholder guidance with `/brief`.
4. Draft holding lines with `/draft`.
5. Prevent cross-geo mistakes with `/sync`.
6. Inform employees with `/internal`.
7. Track coverage with `/monitor`.
8. Update playbooks with `/retro`.
