# Sync

## Mission

Prevent friendly fire and make local-global coordination explicit before it becomes expensive.

## Use this skill when

- the user explicitly asks for `/sync` or the equivalent workflow
- the operating context matches this skill's mission more closely than a generic drafting pass
- the result needs to be stored or reasoned about as part of the Comms-Stack system

## Read before acting

- current and upcoming campaign plans
- `knowledge/sync-calendar/README.md`
- `knowledge/stripe-narrative.md`
- `knowledge/policy-landscape.md` when relevant

## Workflow

1. Choose the mode: calendar, translation, or escalation.
2. Map the overlap in timing, audience, and narrative territory.
3. State whether the conflict is content, politics, timing, or perception.
4. If translating, localize the message for Ireland without weakening the global truth.
5. If escalating, provide options with trade-offs and a recommendation.

## Output contract

Use the relevant repo template when it helps. The output should include:

- conflict map
- localized variant
- coordination brief
- options and trade-offs
- recommended path

## Quality bar

- Name perception risk explicitly.
- Do not assume another team sees the local context.
- Escalation briefs should be short and decisive.

## Escalate immediately when

- exclusive risk
- timing collision
- global content that would undermine a local story
- crisis-adjacent overlap

## Memory update rule

Record meaningful future conflicts in sync calendar memory.

## Do not

- Do not solve political conflicts with wordsmithing alone.
- Do not bury the actual stakes.
