# LinkedIn

## Mission

Use LinkedIn as a serious professional distribution channel without sounding like thought-leadership sludge.

## Use this skill when

- the user explicitly asks for `/linkedin` or the equivalent workflow
- the operating context matches this skill's mission more closely than a generic drafting pass
- the result needs to be stored or reasoned about as part of the Comms-Stack system

## Read before acting

- `knowledge/voice-memory.md`
- relevant narrative, campaign, or placement context
- `knowledge/style-guide.md`

## Workflow

1. Choose the mode: executive voice, commentary, or amplification.
2. Define the one perspective the post should add.
3. Draft for scroll-stopping clarity, not slogan density.
4. Offer hook variants and a comment strategy.
5. Keep the voice anchored to the named author.

## Output contract

Use the relevant repo template when it helps. The output should include:

- post draft
- hook variants
- CTA options
- comment strategy
- claims needing confirmation

## Quality bar

- Avoid cliche leadership language.
- The post should add something the article or report did not already say.
- Respect each author's voice memory.

## Escalate immediately when

- post intersects policy or crisis response
- the requested voice is under-modeled
- the idea is better as media or memo than a post

## Memory update rule

Strong human edits should feed back into voice memory for the relevant author.

## Do not

- Do not write generic founder-bro LinkedIn copy.
- Do not confuse engagement bait with authority.
