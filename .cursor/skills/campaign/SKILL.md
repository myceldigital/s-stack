# Campaign

## Mission

Translate strategy into an execution plan with clear owners, sequence, and risk controls.

## Use this skill when

- the user explicitly asks for `/campaign` or the equivalent workflow
- the operating context matches this skill's mission more closely than a generic drafting pass
- the result needs to be stored or reasoned about as part of the Comms-Stack system

## Read before acting

- current narrative and angles outputs
- `templates/campaign-template.md`
- relevant archive entries in `knowledge/campaign-archive/`

## Workflow

1. State the objective and success condition.
2. Define audience hierarchy and asset requirements.
3. Sequence the launch to avoid self-conflict.
4. Identify dependencies, approvals, and failure points.
5. Leave the operator with a plan that can actually run.

## Output contract

Use the relevant repo template when it helps. The output should include:

- campaign brief
- asset plan
- channel sequence
- owner map
- approval gates
- risk register
- success metrics

## Quality bar

- Prefer one coherent campaign to a pile of disconnected outputs.
- Call out what must happen first.
- If the plan is fragile, say why.

## Escalate immediately when

- global-local coordination risk
- data or policy dependency
- campaign depends on an unready founder or spokesperson

## Memory update rule

Completed campaigns should later be archived with outcomes and lessons.

## Do not

- Do not treat events as separate from campaigns.
- Do not create channel plans detached from narrative logic.
