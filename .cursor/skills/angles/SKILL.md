# Angles

## Mission

Turn one thesis into several credible hooks without diluting the core idea.

## Use this skill when

- the user explicitly asks for `/angles` or the equivalent workflow
- the operating context matches this skill's mission more closely than a generic drafting pass
- the result needs to be stored or reasoned about as part of the Comms-Stack system

## Read before acting

- the current narrative output
- `knowledge/ireland-media-graph.md`
- `knowledge/founder-pipeline.md` if founder protagonists are relevant

## Workflow

1. List the distinct audiences that matter.
2. Choose the best protagonist for each audience.
3. Write the angle as a hook, not a paragraph.
4. Score each angle for novelty, credibility, and timing.
5. Keep one contrarian backup angle in reserve.

## Output contract

Use the relevant repo template when it helps. The output should include:

- audience-angle matrix
- protagonist recommendation
- timing rationale
- outlet fit
- backup angle

## Quality bar

- Avoid making every angle about Stripe.
- A strong angle should imply why this matters now.
- Do not ship ten mediocre hooks when three sharp ones exist.

## Escalate immediately when

- no angle survives skepticism
- all viable angles depend on a single unconfirmed proof point
- protagonist risk is high

## Memory update rule

Update founder or media memory only if an angle test revealed a durable preference.

## Do not

- Do not recycle the thesis word-for-word as every hook.
- Do not force novelty where clarity is the better weapon.
