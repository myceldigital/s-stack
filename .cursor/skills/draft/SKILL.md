# Draft

## Mission

Generate editable first drafts that already reflect real editorial judgment patterns.

## Use this skill when

- the user explicitly asks for `/draft` or the equivalent workflow
- the operating context matches this skill's mission more closely than a generic drafting pass
- the result needs to be stored or reasoned about as part of the Comms-Stack system

## Read before acting

- `knowledge/style-guide.md`
- `knowledge/voice-memory.md`
- the relevant brief, campaign, incident, or narrative file

## Workflow

1. Confirm the draft type and audience.
2. Load style and voice constraints before writing.
3. Write to the minimum useful length for the format.
4. Mark unsupported claims as `UNVERIFIED`.
5. List the edits most likely still required.

## Output contract

Use the relevant repo template when it helps. The output should include:

- working draft
- claims requiring confirmation
- editor note with likely remaining edits

## Quality bar

- Respect banned words and structural preferences.
- Do not produce smooth but empty paragraphs.
- When evidence is thin, write less, not more.

## Escalate immediately when

- draft requires legal, policy, or crisis review
- the requested voice is under-modeled
- the brief lacks enough evidence to draft credibly

## Memory update rule

After human edits, important repeated patterns should be added to `knowledge/voice-memory.md`.

## Do not

- Do not imply the draft is ready to publish.
- Do not hide uncertainty.
- Do not write fake specificity.
