# Data

## Mission

Turn approved aggregate data into trusted narrative evidence without compromising privacy or overclaiming.

## Use this skill when

- the user explicitly asks for `/data` or the equivalent workflow
- the operating context matches this skill's mission more closely than a generic drafting pass
- the result needs to be stored or reasoned about as part of the Comms-Stack system

## Read before acting

- current narrative or campaign context
- `templates/data-report-template.md`
- `knowledge/source-ledger/README.md`
- `knowledge/policy-landscape.md` if the topic intersects regulation

## Workflow

1. Confirm the thesis before touching data outputs.
2. Use only approved aggregate views and explicitly record that constraint.
3. Translate numbers into insights, not just charts.
4. Separate observed trends from speculative explanation.
5. Prepare launch-specific excerpts for different audiences.

## Output contract

Use the relevant repo template when it helps. The output should include:

- data brief
- chart list
- top insights
- caveats
- privacy-review checklist
- outlet-specific excerpts

## Quality bar

- No raw transaction-level data.
- No claim without a source trail.
- No chart without narrative purpose.
- Caveats are part of the product.

## Escalate immediately when

- privacy ambiguity
- aggregation too thin
- policy sensitivity
- numbers that may be directionally true but not publication-safe

## Memory update rule

Record methodologies, caveats, and source references so the next edition compounds rather than restarts.

## Do not

- Do not launder uncertain interpretation into certainty.
- Do not use the data franchise as a PR stunt with weak substance.
