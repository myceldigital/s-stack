# Internal

## Mission

Keep employees informed early enough to trust the system and represent the company responsibly.

## Use this skill when

- the user explicitly asks for `/internal` or the equivalent workflow
- the operating context matches this skill's mission more closely than a generic drafting pass
- the result needs to be stored or reasoned about as part of the Comms-Stack system

## Read before acting

- relevant campaign or incident materials
- `knowledge/internal-comms-log.md`
- `knowledge/style-guide.md`
- `knowledge/voice-memory.md` if leadership voice matters

## Workflow

1. Define what employees need to know before they hear it elsewhere.
2. Separate internal candor from external positioning.
3. Write the clearest useful explanation of what changed, what it means, and what to do.
4. Anticipate likely employee questions.
5. Log the communication after publication.

## Output contract

Use the relevant repo template when it helps. The output should include:

- internal note
- employee FAQ
- Slack-ready summary
- manager talking points if needed

## Quality bar

- Be more candid than external messaging without breaching confidentiality.
- Reduce rumor space.
- If the answer is not known yet, say so.

## Escalate immediately when

- employees may hear news from media first
- sensitive people implications
- crisis requires rapid employee guidance

## Memory update rule

Append major employee-facing comms moments to `knowledge/internal-comms-log.md`.

## Do not

- Do not copy-paste external messaging as the internal note.
- Do not over-reassure when facts are still moving.
