# Landscape

## Mission

Give the operator the smallest accurate briefing that changes today's decisions.

## Use this skill when

- the user explicitly asks for `/landscape` or the equivalent workflow
- the operating context matches this skill's mission more closely than a generic drafting pass
- the result needs to be stored or reasoned about as part of the Comms-Stack system

## Read before acting

- `knowledge/stripe-narrative.md`
- `knowledge/ireland-media-graph.md`
- `knowledge/policy-landscape.md`
- `knowledge/competitor-intelligence.md`

## Workflow

1. Scan for signal changes, not volume spikes.
2. Classify each signal as narrative, relationship, policy, competitor, or crisis-relevant.
3. Identify whether the signal is weak, meaningful, or urgent.
4. Write the morning brief in order of decision value, not chronology.
5. Only update memory if a real new pattern, entity, or conflict emerged.

## Output contract

Use the relevant repo template when it helps. The output should include:

- top developments
- why this matters today
- competitive context
- reporter watch
- policy watch
- recommended actions

## Quality bar

- Never confuse chatter with a trend.
- Mark weak evidence as `weak signal`.
- Flag timing sensitivity explicitly.
- If a reporter appears to be building a story, say why.

## Escalate immediately when

- three or more aligned signals in a short window
- possible crisis or security framing
- reporter behavior suggesting imminent publication
- any signal that could create SF-Dublin friendly fire

## Memory update rule

Append only meaningful deltas to media, competitor, or policy memory.

## Do not

- Do not produce a generic news digest.
- Do not quantify sentiment in a tiny market just because a dashboard could.
- Do not present inference as confirmed reporting.
