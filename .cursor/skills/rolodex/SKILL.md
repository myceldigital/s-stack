# Rolodex

## Mission

Turn fleeting interaction context into reusable relationship intelligence.

## Use this skill when

- the user explicitly asks for `/rolodex` or the equivalent workflow
- the operating context matches this skill's mission more closely than a generic drafting pass
- the result needs to be stored or reasoned about as part of the Comms-Stack system

## Read before acting

- existing relationship notes in `knowledge/ireland-media-graph.md` and `knowledge/partner-tracker.md`
- relevant brief used before the interaction if available

## Workflow

1. Capture what happened factually first.
2. Separate explicit statements from inferred preferences.
3. Update relationship status only if the change is justified.
4. Link the contact to other relevant entities when that linkage changes future comms decisions.
5. Record the best next follow-up, if any.

## Output contract

Use the relevant repo template when it helps. The output should include:

- interaction note
- relationship delta
- new preference or risk note
- linked entities
- follow-up recommendation

## Quality bar

- Prefer durable patterns over ephemeral detail.
- Store why a preference was inferred.
- Keep notes useful for future decisions, not archival completeness.

## Escalate immediately when

- relationship damage
- request for correction or complaint
- new cross-relationship that affects strategy
- evidence that a prior approach felt AI-generated or inauthentic

## Memory update rule

This skill is a memory writer. Be precise.

## Do not

- Do not store gossip.
- Do not overfit one interaction into a permanent rule.
- Do not write vague notes like "good chat".
